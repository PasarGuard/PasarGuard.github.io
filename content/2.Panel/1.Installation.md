  ### پس از نصب

  پس از تکمیل نصب:

  -   لاگ‌هایی را خواهید دید که می‌توانید با بستن ترمینال یا فشار دادن `Ctrl+C` تماشای آن‌ها را متوقف کنید
  -   فایل‌های PasarGuard در `/opt/pasarguard` قرار خواهند گرفت
  -   فایل پیکربندی در `/opt/pasarguard/.env` قابل یافتن است (به بخش [پیکربندی‌ها](#configuration) مراجعه کنید تا متغیرها را ببینید)
  -   فایل‌های داده در `/var/lib/pasarguard` قرار خواهند گرفت
  -   به دلایل امنیتی، داشبورد PasarGuard از طریق آدرس IP قابل دسترسی نیست. بنابراین، باید [گواهی SSL دریافت کنید](https://PasarGuard.github.io/PasarGuard/en/examples/issue-ssl-certificate) و با باز کردن مرورگر وب و رفتن به `https://YOUR_DOMAIN:8000/dashboard/` به داشبورد PasarGuard خود دسترسی پیدا کنید (YOUR_DOMAIN را با دامنه واقعی خود جایگزین کنید)
  -   همچنین می‌توانید از پورت فورواردینگ SSH برای دسترسی محلی به داشبورد PasarGuard بدون دامنه استفاده کنید. `user@serverip` را با نام کاربری SSH واقعی و IP سرور خود جایگزین کنید و دستور زیر را اجرا کنید:

  ```bash
  ssh -L 8000:localhost:8000 user@serverip
  ```

  در نهایت، می‌توانید لینک زیر را در مرورگر خود وارد کنید تا به داشبورد PasarGuard خود دسترسی پیدا کنید:

  http://localhost:8000/dashboard/

  به محض بستن ترمینال SSH، دسترسی به داشبورد را از دست خواهید داد. بنابراین، این روش فقط برای اهداف آزمایشی توصیه می‌شود.
  ### ایجاد کاربر ادمین
  سپس، باید یک ادمین sudo برای ورود به داشبورد PasarGuard با دستور زیر ایجاد کنید

  ```bash
  pasarguard cli admin create --sudo
  ```

  همین! می‌توانید با این اعتبارنامه‌ها به داشبورد خود وارد شوید

  برای دیدن پیام کمک اسکریپت PasarGuard، دستور زیر را اجرا کنید

  ```bash
  pasarguard --help
  ```
