در این صفحه می‌توانید لیستی از تمام متغیرهای محیطی PasarGuard را پیدا کنید. می‌توانید مقادیر را به تمام این متغیرها در فایل .env اختصاص دهید.

::alert{icon="lucide:info"}
  اگر PasarGuard را با روش نصب سریع نصب کرده‌اید، می‌توانید فایل `.env` را در `/opt/pasarguard/.env` پیدا کنید.
::

::field{name="UVICORN_HOST" type="string" defaultValue="0.0.0.0"}
آدرس‌های IP که PasarGuard باید روی آن‌ها گوش دهد.

**توجه:** `0.0.0.0` به معنای تمام آدرس‌های موجود روی ماشین است.
::

::field{name="UVICORN_PORT" type="number" defaultValue="8000"}
پورت که PasarGuard باید روی آن گوش دهد.
::

::field{name="UVICORN_UDS" type="string"}
آدرس سوکت دامنه یونیکس که PasarGuard باید روی آن گوش دهد.

**توجه:** اگر این متغیر مقداردهی شود، متغیرهای `UVICORN_HOST` و `UVICORN_PORT` نادیده گرفته خواهند شد.
::

::field{name="UVICORN_SSL_CERTFILE" type="string"}
مسیر به فایل گواهی SSL.

**مثال:** `/path/to/example.com/fullchain.pem`
::

::field{name="UVICORN_SSL_KEYFILE" type="string"}
مسیر به فایل کلید SSL.

**مثال:** `/path/to/example.com/key.pem`
::

::field{name="XRAY_JSON" type="string" defaultValue="xray_config.json"}
مسیر به فایل JSON پیکربندی xray.
::

::field{name="XRAY_SUBSCRIPTION_URL_PREFIX" type="string"}
پیشوند URL برای لینک‌های اشتراک.

::alert{type="warning"}
اگر این متغیر تنظیم نشود، لینک‌های اشتراک به درستی در ربات تلگرام ارسال نخواهند شد.
::
::

::field{name="XRAY_SUBSCRIPTION_PATH" type="string" defaultValue="sub"}
مسیر برای لینک‌های اشتراک.
::

::field{name="XRAY_EXECUTABLE_PATH" type="string" defaultValue="/usr/local/bin/xray"}
مسیر به فایل اجرایی xray.
::

::field{name="XRAY_ASSETS_PATH" type="string" defaultValue="/usr/local/share/xray"}
مسیر به دایرکتوری فایل‌های دارایی برای xray (فایل‌های `geoip.dat` و `geosite.dat`).
::

::field{name="XRAY_EXCLUDE_INBOUND_TAGS" type="string"}
تگ‌های ورودی که نباید در لیست پروکسی گنجانده شوند و نیازی به مدیریت ندارند.

**مثال:** `"IBOUND_X INBOUND_Y INBOUND_Z"`
::

::field{name="XRAY_FALLBACKS_INBOUND_TAG" type="string"}
اگر از ورودی با چندین fallback استفاده می‌کنید، تگ را اینجا وارد کنید.
::

::field{name="TELEGRAM_API_TOKEN" type="string"}
توکن ربات تلگرام (از [@botfather](https://t.me/botfather) دریافت شده)
::

::field{name="TELEGRAM_ADMIN_ID" type="string"}
ID عددی ادمین در تلگرام (از [@userinfobot](https://t.me/userinfobot) دریافت شده)
::

::field{name="TELEGRAM_LOGGER_CHANNEL_ID" type="string"}
ID عددی کانال لاگ ربات تلگرام.

اگر مقداردهی شود، لاگ‌های مربوط به ربات تلگرام PasarGuard به این کانال ارسال خواهند شد. ربات تلگرام PasarGuard باید ادمین این کانال باشد.
::

::field{name="TELEGRAM_DEFAULT_VLESS_FLOW" type="string"}
flow پیش‌فرض برای پروتکل Vless در ربات تلگرام.

**مثال:** `"xtls-rprx-vision"`
::

::field{name="TELEGRAM_PROXY_URL" type="string"}
برای اجرای ربات تلگرام با پروکسی (در صورت مسدود بودن سرورهای تلگرام روی سرور شما).

**مثال:** `"socks5://127.0.0.1:1080"`
::

::field{name="DISCORD_WEBHOOK_URL" type="string"}
آدرس webhook دیسکورد.
::

::field{name="CUSTOM_TEMPLATES_DIRECTORY" type="string" defaultValue="app/templates"}
دایرکتوری برای فایل‌های قالب.
::

::field{name="CLASH_SUBSCRIPTION_TEMPLATE" type="string" defaultValue="clash/default.yml"}
قالب استفاده شده برای تولید پیکربندی‌های Clash.

**مثال:** [default.yml](https://github.com/PasarGuard/panel/blob/master/app/templates/clash/default.yml)
::

::field{name="SUBSCRIPTION_PAGE_TEMPLATE" type="string" defaultValue="subscription/index.html"}
قالب استفاده شده برای صفحه اطلاعات اشتراک.

**مثال:** [index.html](https://github.com/PasarGuard/panel/blob/master/app/templates/subscription/index.html)
::

::field{name="HOME_PAGE_TEMPLATE" type="string" defaultValue="home/index.html"}
قالب استفاده شده برای صفحه اصلی.

**مثال:** [index.html](https://github.com/PasarGuard/panel/blob/master/app/templates/home/index.html)
::

::field{name="SUB_PROFILE_TITLE" type="string" defaultValue="Subscription"}
عنوان اشتراک در کلاینت.

اگر کلاینت این ویژگی را پشتیبانی کند و کاربر عنوانی برای اشتراک انتخاب نکند، این مقدار به عنوان عنوان اشتراک استفاده خواهد شد.
::

::field{name="SUB_SUPPORT_URL" type="string"}
آدرس تماس پشتیبانی در لینک اشتراک.

**مثال:** `"https://t.me/support"`
::

::field{name="SUB_UPDATE_INTERVAL" type="number" defaultValue="12"}
فاصله زمانی بین به‌روزرسانی‌های خودکار اشتراک (در ساعت).

اگر کلاینت این ویژگی را پشتیبانی کند، اشتراک هر 12 ساعت به‌روزرسانی خواهد شد.
::

::field{name="SQLALCHEMY_DATABASE_URL" type="string" defaultValue="sqlite:///db.sqlite3"}
آدرس پایگاه داده در فرمت SQLAlchemy.

**راهنما:** فرمت‌های موجود و درایورها برای آدرس پایگاه داده را در [مستندات SQLAlchemy](https://docs.sqlalchemy.org/en/20/core/engines.html#database-urls) ببینید.
::

::field{name="WEBHOOK_ADDRESS" type="string" defaultValue="DEFAULT"}
::

::field{name="WEBHOOK_SECRET" type="string" defaultValue="DEFAULT"}
::

::field{name="SUDO_USERNAME" type="string"}
می‌توانید نام کاربری سوپر ادمین را از متغیر محیطی تنظیم کنید.

::alert{type="warning"}
قویاً توصیه می‌شود از CLI PasarGuard برای ایجاد ادمین استفاده کنید و از این متغیر استفاده نکنید.
::
::

::field{name="SUDO_PASSWORD" type="string"}
می‌توانید رمز عبور سوپر ادمین را از متغیر محیطی تنظیم کنید.

::alert{type="warning"}
قویاً توصیه می‌شود از CLI PasarGuard برای ایجاد ادمین استفاده کنید و از این متغیر استفاده نکنید.
::
::

::field{name="DOCS" type="boolean" defaultValue="false"}
فعال کردن مستندات API در `/docs` و `/redoc`.
::

::field{name="JWT_ACCESS_TOKEN_EXPIRE_MINUTES" type="number" defaultValue="1440"}
زمان انقضای توکن دسترسی در دقیقه.

**توجه:** `0` به معنای 'بدون انقضا' است.
::

::field{name="DEBUG" type="boolean" defaultValue="false"}
فعال کردن حالت توسعه.
::

::field{name="VITE_BASE_API" type="string" defaultValue="/api/"}
پیشوند مسیر API برای استفاده در داشبورد (front-end)
::
