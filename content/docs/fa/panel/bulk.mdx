عملیات گروهی برای تغییر چندین کاربر به صورت همزمان است. بعد از ایمپورت کاربران، جابه‌جایی بین گروه‌ها، تغییر محدودیت‌ها یا تعمیر IPهای WireGuard کاربرد دارد.

## عملیات‌های موجود

- **ایجاد کاربران**: ساخت چند کاربر از روی قالب کاربر.
- **گروه‌ها**: اضافه یا حذف کردن گروه برای کاربران انتخاب‌شده.
- **تاریخ انقضا**: اضافه یا کم کردن زمان از کاربران انتخاب‌شده.
- **محدودیت داده**: اضافه یا کم کردن حجم ترافیک.
- **تنظیمات پروکسی**: تغییر تنظیمات پروکسی کاربران انتخاب‌شده.
- **IPهای همتای WireGuard**: تخصیص یا تعمیر IPهای همتا.

## روش امن

1. تب عملیات را انتخاب کنید.
2. تنظیمات عملیات را مشخص کنید.
3. کاربران، ادمین‌ها، گروه‌ها یا فیلتر وضعیت را انتخاب کنید.
4. اول از **پیش‌نمایش** استفاده کنید.
5. خلاصه عملیات را بخوانید.
6. فقط وقتی تعداد هدف‌ها درست بود عملیات را اعمال کنید.

در بعضی صفحات، اگر هیچ هدفی انتخاب نکنید، عملیات می‌تواند روی همه کاربران مطابق فیلتر اجرا شود. قبل از اعمال، هشدار را بخوانید.

## IPهای همتای WireGuard

از **عملیات گروهی > IPهای همتای WireGuard** وقتی استفاده کنید که کاربران به IP همتای WireGuard نیاز دارند.

- وقتی فقط می‌خواهید IPهای ناقص یا نامعتبر تعمیر شوند، **جایگزینی همه IPها** را خاموش بگذارید.
- وقتی می‌خواهید همه کاربران هدف IP جدید بگیرند، **جایگزینی همه IPها** را روشن کنید.
- برای تست، از فیلتر وضعیت یا گروه‌های کوچک‌تر استفاده کنید.
- قبل از اعمال، پیش‌نمایش بگیرید تا تعداد کاربران و ورودی‌های WireGuard اثرپذیر را ببینید.

این کار را بعد از ساخت هسته WireGuard، ایمپورت کاربران یا تغییر محدوده آدرس WireGuard انجام دهید.

## عادت‌های خوب

- قبل از تغییر همه کاربران، با یک گروه کوچک شروع کنید.
- محدودیت داده، تاریخ انقضا و گروه‌ها را همزمان تغییر ندهید.
- فیلترهایی که استفاده کرده‌اید را یادداشت کنید.
- بعد از تغییر بزرگ، [آمار](./statistics) و لاگ گره را بررسی کنید.
