## 系统要求

### 最低要求

- **RAM:** 1 GB
- **CPU:** 1 核心
- **存储:** 10 GB

<Callout type="warning">
**重要提示：** 使用最低配置时，请定期监控磁盘使用情况，尤其是在使用 MySQL 系列数据库时。如果日志或数据库文件占用过多磁盘空间，请考虑升级服务器资源。
</Callout>

### 推荐要求

- **RAM:** 2 GB
- **CPU:** 2 核心
- **存储:** 20 GB

---

## 安装选项

PasarGuard 面板支持多个数据库后端：

- **TimescaleDB** - 针对时间序列数据和分析进行优化（**推荐**）
- **PostgreSQL** - 高级功能和可扩展性
- **MariaDB** - MySQL 的开源替代品
- **MySQL** - 适合生产环境
- **SQLite** - 适合小规模部署和测试

### 安装后

安装完成后：

-   您将看到日志，可以通过关闭终端或按 `Ctrl+C` 停止查看
-   PasarGuard 文件将位于 `/opt/pasarguard`
-   配置文件可以在 `/opt/pasarguard/.env` 找到（请参考[配置](#configuration)部分查看变量）
-   数据文件将放置在 `/var/lib/pasarguard`
-   出于安全考虑，PasarGuard 仪表板无法通过 IP 地址访问。因此，您必须[获取 SSL 证书](https://PasarGuard.github.io/PasarGuard/en/examples/issue-ssl-certificate)并通过打开网页浏览器并导航到 `https://YOUR_DOMAIN:8000/dashboard/` 来访问您的 PasarGuard 仪表板（将 YOUR_DOMAIN 替换为您的实际域名）
-   您也可以使用 SSH 端口转发在没有域名的情况下本地访问 PasarGuard 仪表板。将 `user@serverip` 替换为您的实际 SSH 用户名和服务器 IP，然后运行以下命令：

```bash
ssh -L 8000:localhost:8000 user@serverip
```

最后，您可以在浏览器中输入以下链接来访问您的 PasarGuard 仪表板：

http://localhost:8000/dashboard/

一旦您关闭 SSH 终端，您将失去对仪表板的访问权限。因此，此方法仅推荐用于测试目的。

### 创建所有者账号

使用面板登录页面中的设置表单和一次性临时密钥来创建第一个所有者账号。

1. 打开上一步中的面板登录页面。
2. 在登录页面选择 **所有者访问**。
3. 选择 **创建所有者**。
4. 在服务器上生成临时设置密钥：

```bash
pasarguard cli generate-temp-key
```

5. 将密钥粘贴到设置表单中，输入所有者用户名和密码，然后创建所有者账号。

<Callout type="warning">
临时设置密钥有效期为 5 分钟，并且只能使用一次。如果密钥过期，请生成新的密钥。
</Callout>

如果要将现有管理员设为所有者，请返回登录页面，选择 **所有者访问**，再选择 **将管理员设为所有者**，生成新的临时密钥，然后输入现有管理员用户名和该密钥。只有在当前还没有所有者账号时，此操作才可用。

要查看 PasarGuard 脚本的帮助信息，请运行：

```bash
pasarguard --help
```
