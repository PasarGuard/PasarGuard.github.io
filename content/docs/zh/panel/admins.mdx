管理员是面板操作员。根据分配的角色和权限覆盖，他们可以管理用户、分组、节点、设置或模板。

## 创建管理员

1. 打开 **管理员**。
2. 点击 **创建管理员**。
3. 设置用户名、密码、状态和角色。
4. 如有需要，添加资料标题、订阅域名、支持链接、Telegram ID、Discord Webhook 和备注。
5. 只有该管理员需要与角色不同的限制时，才设置权限覆盖。
6. 保存管理员。

在分配给管理员前，先到 [管理员角色](./admin_roles) 创建或调整角色。

## 所有者访问

第一个所有者通过登录页和临时密钥创建：

```bash
pasarguard cli generate-temp-key
```

1. 在服务器上运行命令并复制临时密钥。
2. 打开面板登录页。
3. 打开 **所有者访问**。
4. 选择 **创建所有者**，输入临时密钥。

如果要把现有管理员变成所有者，打开 **所有者访问**，选择 **将管理员设为所有者**，然后输入临时密钥和管理员账号。

## 管理员表格操作

- **编辑**：修改资料字段、角色、状态、限制和通知。
- **禁用 / 启用**：阻止或恢复管理员账号。
- **重置用量**：重置该管理员管理的用户用量。
- **禁用活跃用户**：禁用该管理员下的所有活跃用户。
- **启用禁用用户**：启用该管理员下已禁用的用户。

这些操作可能影响很多用户，请谨慎使用。

## 权限覆盖

权限覆盖让单个管理员与分配的角色不同。字段留空会继承角色值。覆盖应只用于例外情况，不应作为主要访问管理方式。

## 验证

创建或修改管理员后，用该账号登录一次。确认它能访问预期页面，并且不能访问所有者专用或无关页面。
